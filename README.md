# Sigfrid Portfolio

A modern, responsive personal portfolio website built with HTML, CSS, and JavaScript.

## Project Structure

```
Sigfrid-Portfolio/
├── index.html         # Main HTML file
├── styles.css         # Styling and responsive design
├── script.js          # JavaScript functionality
├── assets/            # Place additional assets here
├── images/            # Place your project images here
└── README.md          # This file
```

## Features

- ✅ Responsive design (works on mobile, tablet, and desktop)
- ✅ Smooth scroll navigation
- ✅ About/Bio section
- ✅ Projects showcase with links
- ✅ Contact section
- ✅ Modern gradient design
- ✅ Scroll animations

## Getting Started

1. Open `index.html` in your browser to view the portfolio
2. Customize the following:

### Personalization

**In `index.html`:**
- Change "Sigfrid" to your name
- Update the subtitle
- Edit your skills in the About section
- Add your projects in the Projects section
- Update contact links (email, GitHub, LinkedIn)

**In `styles.css`:**
- Modify colors in the `:root` section for custom color scheme
- Adjust fonts and spacing as needed

**In `script.js`:**
- Add more interactivity as desired

### Adding Projects

Replace the placeholder project cards in the Projects section:

```html
<div class="project-card">
    <div class="project-image"></div>
    <h3>Your Project Title</h3>
    <p>Description of what you built and technologies used.</p>
    <div class="project-links">
        <a href="project-url" class="project-link">View Project</a>
        <a href="github-url" class="project-link">GitHub</a>
    </div>
</div>
```

## Deployment

### Quick Deploy Options:
- **GitHub Pages**: Push to GitHub and enable Pages in settings
- **Netlify**: Drag and drop your folder
- **Vercel**: Connect your repository
- **Any web hosting**: Upload the files via FTP

## Customization Tips

1. **Add project images**: Place images in the `/images` folder and reference them in the project cards
2. **Change colors**: Edit the CSS variables in `styles.css` (--primary-color, --secondary-color)
3. **Add more sections**: Copy existing sections and modify them
4. **Mobile optimization**: The design is already mobile-responsive
